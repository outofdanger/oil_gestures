# Interaction Spec v1 — жест → действие в сцене

Это **единый источник правды о смысле жестов**. Транспорт (как байты летят между
процессами) описан отдельно в [`integration_contract.md`](integration_contract.md);
этот документ описывает **что жесты означают** в предметной области нефтескважины.

ML, 3D и UI правят этот файл совместно. Любое изменение маппинга — через PR в этот
документ, а не в коде каждого по отдельности.

## Принципы (почему так)

1. **ML отдаёт только сырые жесты.** Он распознаёт `FIST`, `SWIPE_LEFT`,
   `INDEX_SQUEEZE` и т.п. и публикует их по контракту. ML **не знает** состояния
   сцены (что выбрано, какой режим) и поэтому **не вычисляет** доменные команды.
2. **Маппинг «жест → действие» исполняет 3D-приложение.** Только оно знает, какой
   объект выбран, какой сейчас режим и применимо ли действие.
3. **UI ничего не интерпретирует.** Он читает этот документ, чтобы показывать
   человекочитаемые подписи («Свайп вправо → Поворот камеры»), и берёт телеметрию
   (жест, FPS, кадр) из контракта.
4. **Имена жестов заморожены здесь.** Все три стороны кодируют против этих строк.
   Жест может ещё не производиться ML (нет модели) — это не мешает 3D/UI писать
   реакцию на него уже сейчас и тестировать через мок-продюсер
   (`scripts/mock_ml_events.py`).

## Режимы

3D-приложение всегда находится в одном из режимов. Режим определяет 3D, не ML.

| Режим | Когда активен | Что делают свайпы |
| --- | --- | --- |
| `NAVIGATION` | объект не выбран (по умолчанию) | переключение между объектами скважины |
| `CONTROL` | объект выбран | вращение камеры вокруг объекта |

Переход `NAVIGATION → CONTROL`: жест `POINT` выбирает объект под указателем.
Переход `CONTROL → NAVIGATION`: **открытый вопрос** (см. ниже) — предлагается
отдельный жест «снять выбор» либо `POINT` по пустому месту.

## Словарь жестов (frozen)

`produced` = производит ли ML сегодня. `planned` = имя зафиксировано, данные
появятся, когда будет модель/правило; до тех пор эмулируется моком.

| Имя | Канал | Статус | Описание |
| --- | --- | --- | --- |
| `OPEN_PALM` | static | produced | раскрытая ладонь |
| `FIST` | static | produced | кулак |
| `THUMB_UP` | static | produced | большой палец вверх |
| `VICTORY` | static | produced | «виктори» (две пальца) |
| `OK_SIGN` | static | planned | жест «ОК» |
| `POINT` | static | planned | указательный палец (наведение/выбор) |
| `SWIPE_LEFT` | dynamic | planned | свайп влево |
| `SWIPE_RIGHT` | dynamic | planned | свайп вправо |
| `SPREAD` | dynamic | planned | разжатие кисти |
| `CLENCH` | dynamic | planned | сжатие кисти |
| `WRIST_ROTATE_CW` | dynamic | planned | поворот кисти по часовой |
| `WRIST_ROTATE_CCW` | dynamic | planned | поворот кисти против часовой |
| `INDEX_MCP` | cursor | produced | точка указателя (курсор) |
| `INDEX_SQUEEZE` | cursor | produced | щипок указательный+большой (зажатие) |
| `INDEX_RELEASE` | cursor | produced | размыкание указательный+большой |
| `MIDDLE_PINCH` | cursor | produced | щипок средний+большой |

## Маппинг: жест → действие в сцене

Колонка «Исполнитель» — кто реализует логику. `3D` = в 3D-приложении.
`ML` = уже делает рантайм ML (режим курсора, OS-мышь). `UI` = только показывает.

| Жест | Намерение | `NAVIGATION` | `CONTROL` | Исполнитель |
| --- | --- | --- | --- | --- |
| `OPEN_PALM` | стоп / пауза | пауза всего | пауза всего | 3D |
| `POINT` | выбор объекта | выбрать объект под указателем → `CONTROL` | — | 3D |
| `SWIPE_LEFT` | навигация / камера | предыдущий объект | повернуть камеру влево | 3D |
| `SWIPE_RIGHT` | навигация / камера | следующий объект | повернуть камеру вправо | 3D |
| `FIST` | закрыть **задвижку** | — | закрыть задвижку выбранного объекта | 3D |
| `SPREAD` | открыть **вентиль** | — | открыть вентиль выбранного объекта | 3D |
| `CLENCH` | закрыть **вентиль** | — | закрыть вентиль выбранного объекта | 3D |
| `OK_SIGN` | запуск ЭЦН | — | запустить ЭЦН выбранного объекта | 3D |
| `WRIST_ROTATE_CW` | давление + | — | увеличить давление | 3D |
| `WRIST_ROTATE_CCW` | давление − | — | уменьшить давление | 3D |
| `VICTORY` | вкл/выкл режим курсора | переключает режим курсора | переключает режим курсора | ML |

> «задвижка» и «вентиль» в ТЗ — разные элементы: `FIST` управляет задвижкой,
> `SPREAD`/`CLENCH` — вентилем. Если это один элемент, объедините строки.

## Режим курсора (управляется ML напрямую)

Включается/выключается жестом `VICTORY`. Пока включён, ML **сам** двигает
системную мышь и кликает (см. [`gesture_list.md`](gesture_list.md)). 3D-окно как
обычное десктоп-приложение получает эти клики нативно; дополнительно состояние
курсора есть в контракте (`cursor.*`) для отрисовки в UI.

| Действие пользователя | Жесты | Результат |
| --- | --- | --- |
| Левый клик | `INDEX_SQUEEZE` → `INDEX_RELEASE` (короткий) | клик ЛКМ |
| Drag | `INDEX_SQUEEZE`, движение, `INDEX_RELEASE` | зажатая ЛКМ + перетаскивание |
| Правый клик | `MIDDLE_PINCH` | клик ПКМ |
| Перемещение | `INDEX_MCP` | движение курсора |

## Кто что делает (резюме разделения)

- **ML (ты):** распознаёт жесты словаря выше и публикует их по контракту;
  реализует режим курсора. Не знает про сцену, режимы и доменные команды.
- **3D:** держит состояние сцены и режим; исполняет колонки `NAVIGATION`/`CONTROL`.
  Единственное место, где живёт логика «жест → действие».
- **UI:** показывает камеру, текущий жест, FPS, режим курсора; подписи берёт
  отсюда. Логики маппинга не содержит.
